# Experiment: Pushing Docker Images to Docker Hub

## Objective
- The objective of this lab is to learn how to distribute your custom Docker images by pushing them to Docker Hub.
- This allows other developers or deployment environments to pull and use your work.

---

## 1. Prerequisites
- Docker Hub Account: Create a free account at https://hub.docker.com
- Local Docker Image: An existing image (e.g., `my-python-app`)
- Internet Connection: Required to upload image layers

---

## 2. Lab Tasks

### Step 1: Login to Docker Hub
- Authenticate your Docker CLI with Docker Hub credentials.

docker login
Enter your Docker Hub username and password when prompted.
Step 2: Tag the Local Image
Docker Hub naming format: username/repository:tag
docker tag my-python-app <your_username>/my-python-app:v1
my-python-app → Local image name
<your_username> → Your Docker Hub username
v1 → Version tag (optional, default is latest)
Step 3: Push the Image
Upload the image to Docker Hub.
docker push <your_username>/my-python-app:v1
Docker pushes image layers to the registry.
If the repository does not exist, it will be created automatically.
Default visibility is public.
Step 4: Verify the Push
Online Verification
Log in to Docker Hub dashboard.
Check your repository and image tag.
Local Verification
docker pull <your_username>/my-python-app:v1
Confirms the image is available and can be pulled.
